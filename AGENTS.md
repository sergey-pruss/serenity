# Обязательные правила

- По каждой задаче писать реальные тесты; задача сдается только после успешного прохождения тестов по всем условиям задачи и контексту; после принятия задачи и фиксации результата в коммите временные тесты удалять.
- В реализации ориентироваться на оригинал `https://serenity.agency/` как на основной источник правды.
- При верстке по оригиналу (`https://serenity.agency/`) или по скриншотам пользователя добиваться 100% pixel-perfect; доводить результат самостоятельными проверочными снимками до полного визуального совпадения.
- Ничего не придумывать от себя: действовать строго в рамках оригинала; если что-то неясно, сначала уточнять.
- Сообщения коммитов писать на русском языке.
- После завершения задачи выкладывать статику во все три прод-поверхности (один артефакт из репозитория): сервер `/var/www/static` через `bash deploy.sh` (открыто как https://static.serenity.agency и через Nginx как https://serenity.agency), затем отдельно `npx wrangler deploy` для https://serenity.sergeyprus.workers.dev. Подробный порядок команд и проверок — в `CLAUDE.md`, раздел «Деплой».
- Любые изменения в файлах, отдаваемых под `/_sa/` с `Cache-Control: immutable` (CSS/JS), требуют обязательного повышения `?v=` в `html/index.layout.html` и пересборки `npm run build:html`; иначе прод может продолжать отдавать старые версии и создавать ложные регрессии.
- Для `css/*` запрещено использовать пути вида `../_sa/...` внутри `url(...)`; для ассетов статики использовать относительные пути от CSS (`../img/...`), чтобы не получать `/_sa/_sa/...` и `404`.
- После изменений hero-видео или слайдеров обязательна ручная проверка на реальных URL (`serenity.agency`, `static.serenity.agency`, `serenity.sergeyprus.workers.dev`): отсутствие белого экрана в hero, видимость стрелок, корректный дефолт города в меню.
