# Обязательные правила (единый источник)

**Подробный порядок деплоя, архитектура, интеграции:** [`CLAUDE.md`](CLAUDE.md).

**Как устроены URL, роутинг, SEO, аналитика и базовые проверки простым языком:** [`docs/team-handbook.html`](docs/team-handbook.html).

## Тесты и сдача задачи

- Для задачи — **реальные тесты**; результат принимается только после их успешного прохода по условиям задачи и контексту; **временные** тесты после фиксации в коммите **удалять**.

## Источник правды и вёрстка

- Дизайн и контент: ориентир **`https://serenity.agency/`**.
- По оригиналу или скриншотам пользователя — **pixel-perfect**; результат доводить **своими проверочными снимками** до визуального совпадения.
- Не придумывать элементы сверх оригинала; при сомнениях — **уточнять**.

## Коммиты

- Сообщения коммитов — **на русском языке**.

## Деплой: три прод-поверхности (один артефакт)

1. Сервер **`/var/www/static`** — **`bash deploy.sh`** (тот же root для `https://static.serenity.agency` и `https://serenity.agency` через Nginx).
2. Cloudflare Workers — **`npx wrangler deploy`** для `https://serenity.sergeyprus.workers.dev`.
3. Если менялся **маршрутизатор Nginx** (в т.ч. **`nginx/routing.conf`**), дополнительно **`bash scripts/deploy-routing.sh`**. Если менялись продовые конфиги vhost (`nginx/serenity-router.live.conf` и т.п.) — выкладка и reload по шагам в **`CLAUDE.md`**.

## Кэш `immutable` для CSS/JS под `/_sa/`

- После правок этих файлов **обязательно** повысить **`?v=`** в **`html/index.layout.html`** и выполнить **`npm run build:html`**, чтобы клиенты не получали старые бандлы.
- Если затронут оверлей/стили кейсов (в т.ч. `css/css__home-snapshot__overrides.parity-sync.css` и сборка **`case/all`**), синхронно поднимать версии ссылок в соответствующих HTML и выполнять **`npm run build:cases`** там, где это предусмотрено пайплайном — см. **`CLAUDE.md`** и задачу.

## Пути в `url()` внутри CSS

- В **`css/*`** запрещено **`url("../_sa/...")`** — получится **`/_sa/_sa/...`** и **404**. Для статики использовать пути от файла CSS, например **`../img/...`**, **`../js/...`** при необходимости.

## Медиа карточек кейсов (Static / Workers)

- На **`static.serenity.agency`** и **`*.workers.dev`** не использовать **`/cdn-cgi/image/...`** в **`srcset`**, если путь не проверен **`curl -I`** на конкретном хосту (может быть **404** и поломаны превью).
- Превью карточек — прямые URL вида **`/_sa/img/storage__*`** относительно домена.
- После изменений — **`npm run test:case-all`** и визуальная проверка **`case/all`** на трёх поверхностях.

## Прочее при разработке

- **Не множить** стили и скрипты без необходимости.

## После выкладки (опционально)

- При доступе к сети можно выполнить **`npm run test:post-deploy-smoke`** (Playwright, обход трёх origin).
- Дополнительные ручные проверки после деплоя — по задаче и по чеклисту **`CLAUDE.md`** («Анти-регресс чеклист после деплоя»).
