# Главная Serenity — дорожная карта до паритета с продом

Эталон: **https://serenity.agency/**  
Локальная статическая копия для разработки: **`standalone/`** (после `capture:home` → `fix:publish` → `freeze:publish` → `export:standalone`).

---

## Фазы (порядок работ)

| # | Фаза | Цель |
|---|------|------|
| 1 | **Десктоп-паритет вёрстки** | Секции, типографика, отступы на одной эталонной ширине (ниже) совпадают с продом визуально и по структуре. |
| 2 | **Адаптивы** | Повторить брейкпоинты прода (смотреть в DevTools → Computed / вкладка стилей, типичные пороги: ~1024, ~768, ~550 px). |
| 3 | **Поведение** | Скролл (липкая шапка, параллакс и т.д.), ховеры, **бургер** и мобильное меню — отдельные небольшие JS-файлы. |
| 4 | **«Оставить заявку» и формы** | Разметка полей, валидация, отправка на ваш бэкенд/сервис; без лишних продовых трекеров, если не нужны. |
| 5 | **Чистка и оптимизация** | Убрать неиспользуемые Nuxt-чанки; структура `css/`, `js/`, `assets/`; сжатие, при необходимости — сборщик. |

**Порядок внутри фазы 1:** сверху вниз по странице — не перескакивать, иначе отступы между блоками разъезжаются.

---

## Фаза 1 — десктоп-паритет (текущий этап)

### Эталонная ширина

- Основная проверка: **1440px** (или **1920px** — зафиксируйте одну и везде используйте её для «готово/не готово»).
- Второе окно браузера: прод слева, `http://localhost:4180/` справа (или наоборот).

### Чеклист секций главной (порядок сверху вниз)

Отметка: `[ ]` не начато · `[~]` в работе · `[x]` совпадает с продом на эталонной ширине.

- [ ] **Шапка** — логотип, пункты меню, выравнивание, отступы, высота полосы.
- [ ] **Фон** — градиент / точки (`background-gradient`), если визуально отличается от прода.
- [ ] **Герой** — заголовок «Делаем маркетинг лучше», подзаголовок, размеры шрифтов и межстрочные.
- [x] **Видео** — блок Vimeo, скругления, отступы относительно текста (статик: `phase1-overrides.css` + `loading="eager"` в `export-standalone.mjs`).
- [ ] **Кейсы** — полоса карточек / слайдер, отступ до следующего блока.
- [ ] **Клиенты** — блок «Наши клиенты», сетка логотипов, отступы.
- [ ] **«Мы любим маркетинг»** — заголовок, подзаголовок, карточки, кнопка/ссылка «Подробнее о нас» (если есть на проде в этом блоке).
- [ ] **Футер** — колонки, контакты, соцсети, кнопка «Оставить заявку» (внешний вид на этом этапе; логику формы — фаза 4).

### Как фиксировать отличия

1. Открыть прод → DevTools → выбрать элемент → **Computed**: margin, padding, font-size, line-height, max-width.
2. То же на локальной `standalone`.
3. Краткая запись в комментарии к задаче / в этом файле под чеклистом (не обязательно дублировать всё в коде).

### Текущий спринт (первый кусок фазы 1)

1. **[~] Шапка (десктоп 1440px)** — выровнять с продом (меню, логотип, отступы). Пока **без** бургера (это фаза 3).  
   Точечные стили: репозиторий **`assets/phase1-overrides.css`** → при `npm run export:standalone` копируется в **`standalone/site/css/phase1-overrides.css`** и подключается в `index.html`.
2. **Фон героя** — после шапки убедиться, что слой с точками/градиентом совпадает.
3. Далее по чеклисту вниз.

## Целевая структура папки (фаза 5, ориентир)

```
site/
  index.html
  assets/
    css/        # токены, layout, компоненты
    js/         # menu.js, scroll.js, forms.js, …
    img/
    fonts/
    video/      # при необходимости
```

Пока допустимо вести разработку в **`standalone/`**; по мере выноса блоков из Nuxt — переносить в чистую структуру и удалять лишние `/_nuxt` чанки.

## Команды напоминания

```bash
cd /path/to/serenity_lanidng
npm run capture:home && npm run fix:publish && npm run freeze:publish && npm run export:standalone
npm run serve:standalone   # http://localhost:4180
```

После правок в `export-standalone.mjs` или патчах — снова `npm run export:standalone`.
