Gap-анализ: контекстная реклама (SERP)

Сравнение https://serenity.agency/kontekstnaya_reklama/ с органической выдачей топ-20 по 2 запросам, Яндекс и Google, Москва и Санкт-Петербург. Снимок: 2026-05-17. Метод SERP: playwright+curated-baseline, playwright; аудит: fetch HTML. Выдача динамична — ориентир для доработки страницы под топ-10.

Общий вывод: что добавить и изменить для топ-10

На основе SERP-снимка (Playwright, май 2026) и аудита посадочных. Служебные URL Яндекса в сравнении не участвуют.

Контент: приоритетные доработки

  1. Полоса KPI над сгибом — 3–4 цифры (проекты, годы, лиды/кампании, сертификация Яндекс). У большинства лидеров выдачи это есть; у нас отдельного блока нет.
  2. Доверие: логотипы клиентов — горизонтальная лента «нам доверяют» сразу под hero (не путать с кейсами).
  3. Отзывы — отдельная секция с рейтингом/цитатами; кейсы не заменяют social proof.
  4. Гео в первом экране — для Москвы/СПб: «настраиваем и ведём Директ в Москве и Санкт-Петербурге» (не только в FAQ).
  5. Два коммерческих интента — развести акценты:
    • запрос «настройка» — H2/абзац про запуск, аудит, семантику, первые 14–30 дней;
    • запрос «ведение» — H2 про ежемесячное ведение, отчёты, SLA, оптимизацию бюджета.
  6. Сравнение пакетов — таблица «что входит» (Старт / Расширенный / Премиум) в дополнение к слайдеру карточек.
  7. Гарантии и условия — сроки, формат отчётности, что входит в ведение, маркировка (у вас есть в FAQ — вынести видимым блоком).

Gap по матрице блоков (автодетект)

  • Цифры KPI — у ~98% в топ-20; нет у нас в 8/8 выдач.

Объём относительно медианы топ-20

  • число H2: у нас 5, медиана топ-20 — 10, P25 — 9.

Технические и SEO-факторы

  1. Title и H1 — оставить связку «настройка» + «ведение»; в description — Москва/СПб и «Яндекс Директ» без переспама.
  2. Микроразметка — FAQPage и Product/Offer уже есть; после правок FAQ синхронизировать JSON-LD с видимым текстом.
  3. Форма в HTML — инлайн-заявка должна быть в DOM для краулера (не только после JS); проверить видимость полей без скриптов.
  4. Скорость (CWV) — тяжёлый CSS Nuxt: не грузить лишнее above-the-fold; держать актуальный ?v= на /_sa/; LCP hero + не блокировать рендер FAQ/слайдеров.
  5. Внутренние ссылки — блок «Читайте в блоге» (3–4 статьи про контекст/B2B/Директ) с анкорами под кластер.
  6. Индексация — канон на /kontekstnaya_reklama/, в sitemap, без дублей с legacy; после выкладки — переобход в Вебмастере/GSC.
  7. Региональность — при таргете МСК/СПб: явные сигналы в тексте + NAP/офис в футере; при необходимости отдельные посадочные под регион (долгий горизонт).

Что не обязательно для топ-10, но усиливает

  • Калькулятор/квиз бюджета (есть у части конкурентов).
  • Короткое видео (эксперт или кейс) — чаще в Google-выдаче.
  • Онлайн-чат — только при реальной обработке диалогов.

Порядок внедрения (рекомендуемый)

1 KPI + гео в hero → 2 логотипы + отзывы → 3 таблица пакетов + блок «ведение»/«настройка» → 4 гарантии/SLA → 5 блог-тизер и тех. проверка CWV/schema.
