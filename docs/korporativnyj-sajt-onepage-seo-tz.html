Onepage SEO-ТЗ: Корпоративный сайт

https://serenity.agency/korporativnyj_sajt · контур: legacy WordPress · снимок 2026-05-24

Разделение по ролям

Цель — топ-10 по «создание корпоративного сайта» и «разработка корпоративного сайта» (Яндекс и Google). Запросы: SEO-дашборд позиций. Порядок: аудит конкурентов (Сергей Кирин) → макеты (Полина) → тексты и вёрстка (Таня) → статика и SERP gap (Сергей Prus).

Сергей Кирин

Тимлид продукта (услуги) · проектирование и разработка контента

  1. Собрать топ-20 органики по запросам «создание корпоративного сайта» и «разработка корпоративного сайта» — Яндекс и Google, Москва и СПб (8 выдач). Зафиксировать URL конкурентов.
  2. Аудит конкурентов: таблица «URL × блоки» — hero, KPI, этапы, цены, таблица пакетов, кейсы, FAQ, гео, CMS/стек, команда, отзывы, гарантии.
  3. На основе аудита — финальная структура страницы: список H2/H3, какие секции добавить, переименовать или объединить; отдельно развести интенты «создание» и «разработка».
  4. Написать и утвердить тексты коммерческих блоков, этапов работы, подзаголовка hero.
  5. Отобрать 3–4 кейса корпоративных сайтов с метриками (срок, CMS, бизнес-результат) для блока «Ещё кейсы».
  6. Структура и тексты таблицы сравнения пакетов (корпоративный / интернет-магазин / лендинг).
  7. Согласовать ТЗ с Полиной (визуал) и передать финальные тексты Тане в работу.

Полина

Тимлид дизайна

  1. Макет полосы KPI под hero — если текущая вёрстка не закрывает gap по конкурентам (3–4 цифры над сгибом).
  2. Дизайн таблицы сравнения пакетов: десктоп, планшет, мобилка — по структуре от Сергея Кирина.
  3. Визуал карточек кейсов — единый стиль с case/all и другими услугами.
  4. Price-cards: при необходимости — явная иерархия «корпоративный сайт» vs смежные форматы (магазин, лендинг).
  5. Hero: композиция H1 + subtitle + CTA + видео-блок; проверка читаемости на 1440×900 и 375px.

Таня

Помощник · контент и несложная вёрстка

  1. Внести согласованные тексты title, description, H1 и subtitle в JSON/partials (по финальному списку от Сергея Кирина).
  2. Дописать FAQ до 5–7 вопросов по черновику: сроки, стоимость, CMS, поддержка, что входит в разработку.
  3. Подписи к KPI-цифрам и карточкам пакетов — вычитка, типографика NBSP.
  4. Подобрать 3–4 статьи блога для блока «Читайте ещё» — список URL и анкоры для перелинковки.
  5. Сверстать таблицу сравнения пакетов в HTML по макету Полины (когда макет готов).
  6. После go-live: переобход URL в Я.Вебмастер и Google Search Console.

Сергей Prus

SEO · статический контур · архитектура сайта

  1. Полный SERP gap в репозитории (как kontekstnaya-serp-content-gap): автосъёмка выдачи + HTML-отчёт с gap по блокам.
  2. Глубокий gap-анализ: что есть у нас vs топ-20 по каждому блоку; финальные приоритеты P1/P2 для команды.
  3. Стратегия внутренней перелинковки и hub-страниц: какие URL нужны помимо /korporativnyj_sajt для топ-10 в Яндексе и Google (главная, /services, marketing, блог, смежные услуги, case).
  4. Go-live статики: routing.conf (после ОК), deploy-prod/dev, JSON-LD FAQPage/Product, CWV, ?v= на /_sa/.
  5. Реализация правок в репо (HTML, CSS, partials, assemble), test:korporativnyj + test:layout-smoke.
  6. Обновление дашборда позиций и таблицы миграции после выкладки.

Onepage: всё для топ-10 (чеклист работ)

Один документ = ТЗ на SEO и контент. Порядок: P1 → P2 → P3. Полный SERP gap (как у контекстной) — отдельным этапом после съёмки выдачи.

P1 Срочно

  1. Go-live статики + routing.conf (после ОК на конфиг): на prod сейчас legacy без H2 — главный блокер для топ-10.
  2. Title/description: в начале «разработка корпоративного сайта» и «создание корпоративного сайта»; Москва/СПб; CTA без переспама (Яндекс ~12 при 800+ показов, 0 кликов в Google).
  3. Hero: H1 «Разработка корпоративных сайтов» + подзаголовок с «создание корпоративного сайта»; полоса KPI (сайты, отрасли, сроки, награды).
  4. Два коммерческих H2: «Создание корпоративного сайта» и «Разработка корпоративного сайта под ключ» — развести интенты, не только h3 в этапах.

P2 Важно

  1. Таблица сравнения пакетов (корпоративный / интернет-магазин / лендинг) в дополнение к слайдеру price-card.
  2. Кейсы корпоративных сайтов с метриками (срок, CMS, конверсия) — блок «Ещё кейсы» уже есть, усилить SEO-текстом.
  3. FAQ 5–7 пунктов: сроки, стоимость, CMS, поддержка; FAQPage JSON-LD = видимый текст.
  4. Перелинковка: главная, /services, marketing, блог про разработку сайтов.

P3 Дополнительно

  1. После go-live — полный SERP gap (как kontekstnaya/targeting) по двум запросам × Яндекс/Google × Москва/СПб.
  2. Блок «Читайте в блоге» — 3–4 статьи про корпоративные сайты / UX / CMS.
  3. og:image и ассеты в /_sa/img/services/korporativnyj_sajt/…; переобход в GSC и Я.Вебмастер.

Контент и техника

Контент

  • Яндекс (панель): «разработка» ~12.1 (451 пок.), «создание» ~12.1 (372 пок.) — на границе топ-10; SERP СПб: 3 и 8 (снимок 2026-05-19), Москва: 13 и 28.
  • Google: страница вне топ-20 по обоим запросам — после статики усилить title/H2 и внутренние ссылки.
  • Статическая сборка в репо: 8 H2, FAQ, кейсы, награды, price-cards — выше legacy на prod (0 H2, h1 «корпоративный сайт»).

Техника

  • Канон: https://serenity.agency/korporativnyj_sajt (без слэша).
  • Product/Offer schema на price-card — проверить цены и image на /_sa/.
  • Форма заявки в DOM (#desktop-order-popup); CWV после Nuxt CSS bundle.
  • 301 с /korporativnyj_sajt/ при go-live; sitemap lastmod после деплоя.

Миграция на статику
